# Plan: Yazi-inspired Openers System for FTB

## Overview
Implement a configurable file opener system. All actions use `Ctrl+B` prefix mode commands (consistent with existing system), except Enter which opens files with default opener.

## Keybindings

| Trigger | Action |
|---------|--------|
| `Enter` on file | Open with default system opener (rules-based) |
| `Ctrl+B` → `e`/`v`/`vim`/`editor` | Open with built-in editor (**existing**) |
| `Ctrl+B` → `op`/`open` | Show opener picker dialog (**new**) |
| `Ctrl+B` → `ow`/`openwith` | Manual input opener name (**new**) |

## Architecture

### Layer 1: Opener Registry (`[opener]` in config)
```json
{
  "opener": {
    "openers": {
      "edit": [{"run": "", "block": true, "desc": "内置编辑器"}],
      "open": [{"run": "xdg-open %s", "orphan": true, "desc": "系统默认程序"}],
      "code": [{"run": "code %s", "orphan": true, "desc": "VS Code"}],
      "qtcreator": [{"run": "qtcreator %s", "orphan": true, "desc": "Qt Creator"}]
    },
    "rules": [
      {"name": "*", "use": ["open"]},
      {"mime": "text/*", "use": ["edit", "open"]},
      {"name": "*.{pro,cmake}", "use": ["qtcreator", "edit", "open"]}
    ]
  }
}
```

### Layer 2: Rule Matching
- Priority: exact extension glob > MIME type
- `%s` replaced with file path

### Layer 3: Command Dispatch
Enter → default opener | Ctrl+B commands → specific actions

## New Files

### 1. `include/FTB/OpenerManager.hpp`
```cpp
struct Opener {
    std::string run;           // Command with %s placeholder
    bool block = false;        // Block until command exits
    bool orphan = false;       // Background process
    std::string desc;          // Description for picker
};

struct OpenRule {
    std::string name;          // Glob like "*.pro" or MIME like "text/*"
    std::string mime;
    std::vector<std::string> use;  // Opener names in priority order
};

class OpenerManager {
public:
    static OpenerManager& Instance();
    void LoadConfig(const nlohmann::json& j);
    std::vector<std::pair<std::string, Opener>> MatchOpeners(const std::string& filename) const;
    std::optional<Opener> GetDefaultOpener(const std::string& filename) const;
    bool Execute(const Opener& opener, const std::string& filepath, ftxui::ScreenInteractive* screen = nullptr);
    std::vector<std::string> GetOpenerNames() const;
    std::optional<Opener> GetOpener(const std::string& name) const;
};
```

### 2. `src/FTB/OpenerManager.cpp`
- Load opener definitions from JSON config
- Rule matching: extension glob → MIME fallback
- Process execution: block (WithRestoredIO + system) / orphan (fork+exec)
- %s placeholder replacement

### 3. `src/UI/OpenerPickerDialog.cpp`
- Picker dialog showing matched openers with descriptions
- Arrow keys to select, Enter to confirm, Escape to cancel

### 4. `src/UI/OpenerInputDialog.cpp`
- Text input dialog for manual opener name
- Tab completion from registered opener names
