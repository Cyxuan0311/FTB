# 多标签页分栏管理 — 实现计划

## 概述

为 FTB 添加多标签页支持，允许用户同时浏览多个目录，每个标签页独立保存浏览状态。按 `t` 打开新建标签弹窗输入路径，按 `[`/`]` 切换标签。

---

## 架构设计：Snapshot-on-switch

核心策略：**不重构 MainState 的所有调用点**。所有现有代码继续直接读写 `MainState` 的字段（`currentPath`, `selected`, `filteredContents` 等）。切换标签时，通过 `saveTabState()` / `loadTabState()` 在 Tab 结构体与 MainState 之间快照同步。这样：
- 现有代码零修改（Navigation.cpp, FileListRenderer.cpp, EventHandler.cpp 等不变）
- 只在切换点做 save/load
- 新增的 TabManager 和 NewTabDialog 遵循已有模式

## 数据结构

### Tab（per-tab 状态快照）

```cpp
struct Tab {
    // 浏览状态（与 MainState 对应的字段）
    std::string currentPath;
    int selected = 0;
    int hovered_index = -1;
    int current_page = 0;
    int total_pages = 1;
    std::vector<std::string> allContents;
    std::vector<std::string> filteredContents;
    std::set<int> batch_selected;
    std::string searchQuery;
    bool search_mode = false;

    // 独立排序模式
    SortMode sortMode = SortMode::NameAsc;
    bool useGlobalSortMode = true;

    // 独立导航历史
    DirectoryHistory directoryHistory;

    // 独立缓存
    std::string cached_current_path_for_entries;
    std::string cached_parent_path;
    std::string cached_parent_display;
    std::string cached_canonical_path;
    int cached_parent_selected = -1;
    std::vector<FileManager::DirEntryInfo> cached_parent_entries;
    std::vector<FileManager::DirEntryInfo> cached_current_entries;

    std::string displayName() const;  // basename of currentPath
    void init(const std::string& path);
    void refreshContents();  // 切换时刷新目录
};
```

### TabManager

```cpp
class TabManager {
public:
    int createTab(const std::string& path);   // 返回新标签索引
    bool closeTab(int index);                  // 最后一个标签返回 false
    void switchTo(int index);                  // clamp 到有效范围
    void nextTab();                            // 循环切换
    void prevTab();
    Tab& active();
    int activeIndex() const;
    int count() const;
    bool canClose() const;
private:
    std::vector<Tab> tabs_;
    int activeIndex_ = 0;
};
```

### MainState 变更

```cpp
struct MainState {
    // ... 现有字段全部保留 ...
    TabManager tabManager;

    void saveTabState();    // MainState → active Tab
    void loadTabState();    // active Tab → MainState
    SortMode currentSortMode() const;  // 返回当前标签的排序模式
};
```

## 关键设计决策

### 剪贴板策略
**保持全局 ClipboardManager 不变。** 剪贴板存储的是绝对路径，切换标签后 Ctrl+V 可以直接粘贴到新标签的目录。这自然实现了跨标签复制/移动，无需额外逻辑。"每标签独立剪贴板"作为后续增强。

### 排序模式
通过 `Tab::sortMode` + `Tab::useGlobalSortMode` 标志实现。首次打开标签使用全局配置，用户在标签内修改排序后自动切换为 per-tab 模式。`FileManager::getDirectoryEntries()` 添加接受 SortMode 参数的重载。

### 标签栏
使用 FTXUI 的 `hbox` + 文本渲染，活跃标签高亮（TC("selection_bg")/TC("selection_fg")），非活跃标签暗色。仅 tabs > 1 时显示，不浪费屏幕空间。

## 验证方案

1. 启动 FTB → 按 `t` → 弹窗出现 → 输入有效路径 → Enter → 新标签创建，标签栏显示
2. 按 `]` → 切换到下一个标签，文件列表更新；按 `[` → 切换回上一个
3. 在标签 A 中导航到深层目录，切换到标签 B 再切回 A → A 的位置保持
4. 在标签 A 中修改排序为 SizeDesc，切换到标签 B → B 仍使用默认排序
5. `Ctrl+B` → `ct` → 关闭当前标签；最后一个标签 → 提示 "Cannot close the last tab"
6. 标签栏鼠标点击 → 切换到对应标签
7. `Ctrl+B` → `nt` → 打开新建标签弹窗（与 `t` 效果相同）
8. 输入不存在的路径 → 显示错误信息
9. 打开 Vim 编辑器时按 `]` → 无反应
10. 编译通过：`cmake --build build`
